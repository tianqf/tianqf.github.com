实 例一 :共 享一个 文件 夹 /common, SMB 服 务的 工作 组必 须是 SMBtest 一个成员,/common 只能对 ex.com 域里的客户端使用。wy 用户可以读写访问/common,smb 用户密码是 P@33w0rd 配置思路 端 口 号 : 137/udp(netbios-ns),138/udp(netbios-dgm),139/udp(netbios- dgm),139/tcp(netbios-ssn), 445/tcp(Microsoft-ds) 包名: samba,samba-common,samba-client 主配置文件: /etc/samba/smb.conf 守护进程名: smb 本地用户&&samba 用户: useradd wy && smbapasswd –a wy 权限: service 写权限、filesystem 写权限、selinux 权限 1:# nmtui 配置网卡 ip,配好 ip 保存 2:# service network restart 重新启动网卡服务 3:# ifconfig | grep inet 验证 ip 是否配置好 4:# hostname wy.ex.com 配置主机名(立即生效) 5:# echo wy.ex.com > /etc/hostname 配置主机名(永久生效) 6:# cat /etc/hostname 验证是否主机名配置 7:# echo 172.16.7.1 wy.ex.com >> /etc/hosts 在 hosts 文件中追加一行,能够解析这台主机 8:# rpm –q samba samba-client samba-common 查包看是否已经安装 9:# yum –y install samba samba-client samba-common 装包 10:# mkdir /common 创建 common 目录 11:# useradd wy 添加用户 wy 12:# smbpasswd –a wy 添加 samba 用户,要求输入 2 次密码,密码是不可见的 13:# pdbedit –L 确认已经创建好 smb 用户 14:# ls –ld /common 可以看出 /common 文件夹的权限,所以 wy 这个用户默认没有写入权限 15:# setfacl –m u:wy:rwx /common 为了安装所以就用 acl 控制只给 wy 这个用户 rwx 权限,并且用 getfacl 查看权限 16:# ls –ldZ /common 查看 selinux 的目录权限 由于 samba 配置文件写了,要求 selinux 目录权限为 samba_share_t,所以要修改目录 selinux 权限,下面这条命令在 smb.conf 配置文件里面有这条命令: 17:# chcon –R –t samba_share_t 18:# rpm –qc samba-common 查看 samba 服务的配置文件,一般都是以.conf 结尾的是配置文件 19:# grep –E ‘workgroup|hosts’ /etc/samba/smb.conf | grep –v ‘#’ 到默认情况工作组和允许网段访问的可以看 20:# vim /etc/samba/smb.conf 修改配置文件 workgroup = SMBtest hosts allow = 172.16. [common] path = /common browseable = yes writab = yes 21:# service smb restart 重启 samba 服务 22:# chkconfig smb on 开机自动启动 23:# systemctl status smb.service 查看 samba 系统状态 24:# firewall-cmd –permanent –add-service=samba 开启 samba 服务的防火墙 25:# service firwalld restart 重启防火墙 Client Linux: 26:# yum –y install samba-clinet 用 yum 源安装 samba 客户端包(前提需要配置好yum) 27:# smbclient –L //172.16.7.1 –U wy%P@33w0rd windows: Win+R 输入172.16.7.1
